# Проект

В ходе прохождения курса студенты выполняют курсовой проект - веб-сервис. 
Поскольку курс не связан с frontend-ом, то требования к визуальной части выдвигаются минимальные: 
наличие примитивных html-форм для демонстрации работы api-сервера.
Внешний вид html-форм не оценивается, но может быть выполнен студентом по личной иницативе.

## Цель и назначение проекта
Выполнение курсового проекта предназначено для применения и закрепления студентами теоретических знаний, полученных на лекциях,
наработка опыта разработки и формирование профессиональных компетенций в области разработки веб-сервисов.

## Общие требования к проекту

### Технические
1. REST-api сервис (GET, POST, PUT, DELETE, OPTIONS запросы)
1. Наличие авторизации пользователей
1. Хранение данных в реляционной СУБД (предпочтительно PostgreSQL)
1. Еndpoint, взаимодействующий с клиентами протокол WebSocket
1. Микросервисная архитектура с взаимодействием по gRPC
1. Наличие в проекте unit и benchmark тестов

### Организационные
1. Проект должен выполняться в рамках курса   
Запрещается использовать заранее выполненные (в том числе частично) проекты из других других компаний, образовательных проектов и т.п.
1. Проект выполняется индивидуально
1. Запрещается в проекте использовать фрейморки, но разрешается использовать библиотеки

## Формат оценивания
Проект декомпозируется на части, к каждой части предъявляется ряд требований. Каждое требование оценивается определённым количеством баллов. 
При частичной реализации студент получает баллы только за тот функционал, который реализован.
При неполной реализации требования преподаватель в праве оценить функционал количеством баллов, 
не превышающим максимальный балл за конкретное требование.   

Рассмотрим на примере.   
Требования: регистрация пользователя (5 баллов), обновление профиля (3 балла), загрузка аватарки пользователя (5 баллов).    
Если проект, разработанный студентом, удовлетворяет всем требованиям и студент может это продемонстрировать, то студент получает 13 баллов. 
Если студент реализовал только регистрацию пользователя, то студент получает 5 баллов.

### Проект «Ярмарка торговых роботов»
**Общая идея**   
Пользователи регистрируются в сервисе. 
Они могут создавать торговых роботов, добавлять их в избранное или добавлять в избранное роботов других пользователей.
При создании робота указывается цена покупки, цена продажи, планируемая доходность, а также временное окно, в котором будет торговать робот.
В процессе работы, доходность робота будет изменяться. По окончанию работы, роботу присваивается фактическая доходность.
На странице с роботом открывается WebSocket-соединение, которое показывает в режиме онлайн текущую доходность робота и кол-во активированных роботов в избранном всех пользователей.
Робот подключается к торговой системе по средствам gRPC
